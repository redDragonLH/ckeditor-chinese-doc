comment / 注释

CKEDITOR.dom.comment

代表一个DOM注释节点。

var nativeNode = document.createComment( 'Example' );
var comment = new CKEDITOR.dom.comment( nativeNode );

var comment = new CKEDITOR.dom.comment( 'Example' );

属性

$

类型：对象

保存的是原生dom对象

var element = new CKEDITOR.dom.element( 'span' );
alert( element.$.nodeType ); // '1'

type

类型：数字

节点类型。 这是一个设置为 CKEDITOR.NODE_COMMENT 的常量值。

默认为CKEDITOR.NODE_COMMENT

CKEDITOR.NODE_COMMENT 常量默认为 8

静态属性

useCapture : Boolean

方法

constructor

constructor( comment, [ ownerDocument ] ) → comment

创建一个评论类实例。

    参数

         comment ：对象| 字符串

        [ownerDocument] : 文档

在创建新节点的情况下将包含节点的文档。 默认为当前文档。

返回

comment

覆盖：CKEDITOR.dom.node.constructor

appendTo( element ) → element

使此节点成为另一个元素的子节点。

var p = new CKEDITOR.dom.element( 'p' );
var strong = new CKEDITOR.dom.element( 'strong' );
strong.appendTo( p );

参数

    element : element

         此节点将被追加到的目标元素。

Returns

     element

         目标元素。

capture()

在支持目标上的捕获阶段下注册事件处理程序。

clearCustomData()

删除存储在此对象中的任何数据。 为了避免内存泄漏，我们必须确保在不再需要对象之后没有剩余的引用。

clone( [ includeChildren ], [ cloneId ] ) → node

克隆这个节点

注意：由{setCustomData}设置的值在克隆中不可用。

参数

     [ includeChildren ] : Boolean

        如果为 true，则所有节点的子节点将被递归克隆。

        默认为 false

     [ cloneId ] : Boolean

         ID属性是否应该被克隆。

        默认为 false。

返回

     node

         克隆的节点

define( name, meta )

预定义特定事件名称的一些内部属性。

参数

     name :string

         事件名称

     meta : Object

         属性

         [ errorProof ]

             事件触发是否应该捕获每个侦听器调用引发的错误。

             默认为 false
